Information collection

with Metasploit

Information collection is very important for the early stage of penetration test, because only when we have enough information about the target website or host, we can better detect its vulnerability. As the saying goes, the more information you got, the more chance you will success. It's just that knowing yourself and knowing your enemy ,and you need not fear the result of any battle.

#1 Preparatios:

KALI LINUX(used as attacker) IP:192.168.0.108

WINDOWS 7(used as target1) IP:192.168.0.105

Metasploitable2(used as target2) IP:192.168.0.110

metasploit framework(in KALI)

Of course, The network between attacker and target should be interconnected

After this, we could start.

#2 Collect host information based on TCP protocol

#2.1 Collect with nmap and arp_sweep

Use "db_nmap-sv 192.168.1.1" for nmap scanning

#2.2 ARP scanning

Use "search arp_sweep" to find the module

Use the "use" command to load the module

Use the "show options" command to view configuration information

Configure the IP address and thread number of rhost and run the scan module

#2.3 scan TCP port with half connection

Use search to find the module and use the "use auxiliary/scanner/portscan/syn" command to load the module

View the configuration information, configure the IP address, thread number and port range of rhost and run the scanning module

#2.4 Password sniffing with psnuffle

Old routines: Use search to find the module and use the use command to load the module. Run the scan module after viewing the configuration information (no files need to be configured)

Use LFTP command in another terminal to connect with target machine and execute a command.

Go back to the terminal and find that the password has been sniffed

#3 Collecting host information based on SNMP protocol

Use search to find snmp_enum and mount it.

After configuring rhosts, you can run it.

"use auxiliary/scanner/snmp/snmp_enum"

Network card information, IP address, system name have all been scanned.

#4 Collecting information based on SMB protocol

#4.1 Scan version number based on SMB protocol using "smb_version"

Module: auxiliary/scanner/ smb/smb_version

Configuration: you need to configure rhosts. You can add multiple IP addresses, separated by spaces

The version number of windows was successfully scanned out.

#4.2 Use smb_enumshares to scan shared files based on SMB protocol (account and password required)

Add a shared file under the target

Module: auxiliary/scanner/smb/smb_enumshares

Configuration: Need to configure RHOSTS, SMBUser, SMBPass

After running, the shared folder "target" is scanned out

#4.3 Scan system user information (SID enumeration) using smb_lookupsid

Module: auxiliary/scanner/smb/smb_lookupsid

Configuration: Need to configure RHOSTS, SMBUser, SMBPass

Target users are scanned out.

It is worth noting that the rid of the system administrator is 500, while the rids created later are all 1000+.

#5 Collecting information based on SSH protocol

#5.1 View version information of SSH service

Module: auxiliary/scanner/ssh/ssh_version

Configuration: you need to configure rhosts. You can add multiple IP addresses, separated by spaces

The version number of SSH is successfully scanned out. With this version number, you can find the vulnerability of this version on Baidu/Google, and then exploit it

#5.2 Brute-force cracking of SSH

Module: auxiliary/scanner/ssh/ssh_login

Configuration: RHOSTS, dictionary file, VERBOSE (visualization)

#6 Information collection based on FTP protocol

#6.1.1 View version information of FTP service

Module: auxiliary/scanner/ftp/ftp_version

Configuration: you need to configure rhosts. You can add multiple IP addresses, separated by spaces

The version number of FTP is successfully scanned out. With this version number, you can find the vulnerability of this version on Baidu / Google, and then realize vulnerability attack

For example, vsftp2.3.4 has a back door, also known as smiley face vulnerability

#6.1.2 Exploit the version vulnerability of FTP protocol

Module: exploit/unix/ftp/vsftpd_234_backdoor

Configuration: RHOSTS

It can be seen from the query of whoamI that the permission of root account has been obtained

#6.2 FTP anonymous login scan

Module: auxiliary/scanner/ftp/anonymous

Configuration: RHOSTS

As you can see, the version number is also scanned

#6.2 Brute-force cracking of ftp

Module: auxiliary/scanner/ftp/ftp_login
