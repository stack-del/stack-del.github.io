2.   基于 tcp协议收集主机信息

2.1 使用nmap 和 arp_sweep收集

使用db_nmap -sV 192.168.1.1进行nmap扫描

 

2.2 ARP 扫描

使用search arp_sweep找到模块

使用use命令装载该模块

使用show options命令查看配置信息

配置Rhost的ip地址和线程数并且运行扫描模块

 

2.3 使用半连接的方式扫描TCP端口

使用search找到模块，使用use命令装载该模块

查看配置信息，配置Rhost的ip地址、线程数和端口范围并且运行扫描模块

 

2.4 使用psnuffle进行密码嗅探

老套路：

使用search找到模块，使用use命令装载该模块。

查看配置信息（发现无需配置任何文件）后运行扫描模块

在另一个终端中使用通过lftp命令与靶机相连，执行一个命令

回到刚刚的终端发现密码已经被嗅探到

 

3. 基于 SNMP 协议收集主机信息

使用search查找snmp_enum并且装载，配置RHOSTS后使用

use auxiliary/scanner/snmp/snmp_enum

网卡的信息，ip地址，系统名称…都已经被扫描出来

 

4. 基于 SNMP 协议收集信息

4.1使用 smb_version 基于 SMB 协议扫描版本号

模块：auxiliary/scanner/smb/smb_version

配置：需要配置RHOSTS ，可添加多个ip地址，中间要有空格隔开

Windows的版本号成功被扫描出来

 

4.2使用 smb_enumshares 基于 SMB 协议扫共享文件（需要账号、密码）

 

在靶机下添加一个共享文件

 

使用到的模块：auxiliary/scanner/smb/smb_enumshares

配置文件说明：需要配置RHOSTS，SMBUser，SMBPass

运行之后，共享文件夹target被扫描了出来

 

4.3使用 smb_lookupsid 扫描系统用户信息（SID 枚举）

需要用到的模块：auxiliary/scanner/smb/smb_lookupsid

配置文件说明：需要配置RHOSTS，SMBUser，SMBPass

Target用户被扫描出来

值得注意的是，系统管理员的RID为500，而后来创建的RID均在1000+

 

5. 基于 SSH 协议收集信息

5.1查看 ssh 服务的版本信息

模块：auxiliary/scanner/ssh/ssh_version

配置：需要配置RHOSTS ，可添加多个ip地址，中间要有空格隔开

ssh的版本号成功被扫描出来，有了该版本号就可以在baidu/google上查找该版本的漏洞，进而实现漏洞攻击

 

5.2对 SSH 暴力破解

模块：auxiliary/scanner/ssh/ssh_login

配置信息：配置RHOSTS，字典文件，VERBOSE（可视化）
